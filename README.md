## 📖 Описание

**Course Creator** — это учебный курс по архитектуре программных систем для IT-команд. Курс рассчитан на **~10 часов обучения** и содержит структурированные материалы по ключевым темам: от базовых принципов разработки до документирования архитектурных решений.

Курс включает:
- 📘 Теоретические модули с примерами
- 📊 Диаграммы, иллюстрации и схемы
- 🔗 Связанные ссылки на дополнительные материалы
- 📝 Практические задания для закрепления (в процессе разработки)

## 🎯 Целевая аудитория

- Разработчики (Junior+ / Middle), желающие прокачать архитектурное мышление
- Технические лиды и тимлиды
- Системные архитекторы
- Команды, переходящие на микросервисную архитектуру

## 🚀 Быстрый старт

### 1. Клонирование репозитория

```bash
git clone https://github.com/DAndreev91/course_creator.git
cd course_creator
```

### 2. Просмотр материалов

Все материалы курса находятся в корневой директории в формате Markdown. Рекомендуется начинать изучение с файла [final_course_ready.md](final_course_ready.md), который содержит полную структуру курса и ссылки на все модули.

### 3. Работа с Python-окружением (опционально)

Проект использует [uv](https://github.com/astral-sh/uv) для управления зависимостями:

```bash
# Установка uv (если ещё не установлен)
pip install uv

# Установка зависимостей
uv sync

# Активация виртуального окружения
source .venv/bin/activate  # Linux/Mac
# или
.venv\Scripts\activate  # Windows
```

## 📖 Рекомендации по изучению

1. **Для новичков**: начните с Части 1, двигайтесь последовательно.
2. **Для практиков**: выберите интересующий блок (например, Часть 2 для проектирования).
3. **Для команд**: используйте курс как основу для внутреннего обучения — проходите по 1-2 модуля в неделю с обсуждениями.

## 🤝 Как внести вклад

Мы приветствуем вклад в развитие курса! Вы можете помочь:

1. **Сообщить об ошибке** — создайте [Issue](https://github.com/DAndreev91/course_creator/issues) с описанием проблемы.
2. **Предложить улучшение** — опишите идею в Issue или сразу создайте Pull Request.
3. **Добавить практические задания** — дополните модули практическими кейсами.
4. **Перевести материалы** — на другие языки (если есть такая потребность).

### Процесс контрибьюции

1. Форкните репозиторий
2. Создайте ветку для ваших изменений (`git checkout -b feature/your-feature`)
3. Внесите изменения и закоммитьте их (`git commit -m 'Add some feature'`)
4. Отправьте в свой форк (`git push origin feature/your-feature`)
5. Откройте Pull Request в основной репозиторий

## 📋 Требования к файлам

- Все файлы курса должны быть в формате Markdown (`.md`)
- Диаграммы и иллюстрации размещайте в папке `knowledge/` (ссылайтесь на них из модулей)
- Для схем используйте Mermaid или ссылки на внешние ресурсы (Draw.io, Excalidraw)
